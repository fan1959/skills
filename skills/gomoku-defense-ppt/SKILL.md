# Gomoku Defense PPT

从 SCU 答辩模板自动生成 Gomoku 五子棋项目答辩 PPT。

## 设计模式

本 skill 采用 **Generator** 模式：
- 读取 SCU 模板 PPT（30 页、空白版式、手绘设计）
- 按四段结构填充文本框架
- 删除 11 页不适用内容
- 输出 20 页定稿

## Gotchas

- **模板用「空白」layout**：必须用 `slide.shapes[N].text_frame` 直接编辑，不能用 `slide.placeholders`
- **删页必须倒序**：`sorted(pages, reverse=True)`，否则后续 index 错乱
- **装饰说明文字保留**：模板中的颜色板式、设计说明是模板设计意图，不要当占位符清掉
- **不要改模板结构**：只填充文本 + 删页，不加新 shape 和动画

## 工作流

```text
生成进度：
- [ ] 步骤 1：确认项目信息
- [ ] 步骤 2：打开模板，分析文本框架
- [ ] 步骤 3：填充四段内容
- [ ] 步骤 4：删除不适用的 11 页
- [ ] 步骤 5：验证页数和内容
- [ ] 步骤 6：保存输出
```

### 步骤 1：确认项目信息

收集关键信息（如果用户未提供则询问）：
- 项目名称（默认 "Gomoku 五子棋"）
- 技术栈（Qt 5.14.2 + MinGW + SQLite + TCP Socket）
- 项目亮点（3-5 个）
- 答辩人信息

### 步骤 2：打开模板

模板路径：`assets/四川大学答辩PPT.pptx`（30 页、16:9、空白版式）

用 `python-pptx` 加载，遍历所有 slide 的 `shapes` 收集文本框架。

### 步骤 3：填充内容

按四段结构填充：

1. **自我介绍**（1-2 页）：姓名、专业、方向
2. **项目介绍**（6-8 页）：背景、目标、架构、技术栈、功能模块、通信协议、数据库设计
3. **项目亮点**（4-6 页）：分层架构、网络对战、段位系统、棋盘渲染、测试覆盖
4. **总结**（2-3 页）：收获、不足、展望、致谢

每个文本框架用中性、朴实的语言填充，每段 1-3 句话。

### 步骤 4：删除不适用页

删除 11 页模板自带但不适用的内容（倒序删除）。

### 步骤 5：验证

- 总页数 = 20
- 四段结构完整
- 无残留占位符
- 文本不超出框架

### 步骤 6：保存

输出到用户指定路径，默认 `Gomoku_答辩PPT_V1.pptx`。

## 输出要求

- 20 页 .pptx 文件
- 纯文字内容，无 AI 套话
- 中文标点，宋体/黑体
- 每页 1-3 句话，段落短小
